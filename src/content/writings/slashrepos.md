## /til

- topic based subdirectories
- readme per topic
- file per concept, tool, or workflow

```
til/
├── devops/
│   ├── readme.md
│   ├── using-ai-for-incident-response.md
│   └── ...
├── pyspark/
│   ├── readme.md
│   ├── optimizing-dataframes.md
│   └── ...
└── ...
```

- Similiar: /notes

## /awesome

Similiar to /bookmarks. personal awesome list of resources, tools, and so on. For example, a `python.md` file with a curated list of Python libraries and resources.

## /snippets

Collection of code snippets, one per file, organized by topic. For example, a `python/` directory with files like `read_csv.md` containing a code snippet for reading CSV files in Python.

## /bookmarks

Personal bookmark collection, organized by topic. For example, a `webdev.md` file with links to useful web development resources.

## /log

Journal or log of things that happen in my life, both personal and professional. For example, a `2026-04.md` file with entries for each day of April 2026.

## /dotfiles

Developer machine setup files as e.g. `.bashrc` or other configuration files for tools and applications. For example, a `.vscode` directory with a `settings.json` file that contains my personal VS Code settings.

Related: /dotinfra for infrastructure as code (IaC) and similar configuration for homelab, ...

## /recipes

Non-dev related. Actually cooking recipes, home improvement guides, and so on.

## /dotinfra

Like dotfiles but for IaC and similar configuration. For example, Terraform modules, Ansible playbooks, and so on.

## /wiki

Brain dump or wiki-alike knowledge base. For example, a `programming/` directory with files like `oop.md` containing an explanation of object-oriented programming concepts.
